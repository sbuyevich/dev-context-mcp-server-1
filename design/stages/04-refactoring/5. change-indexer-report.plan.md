# Indexer Change Summary

## Summary
Change each index run summary to list exact package changes as `Added`, `Updated`, and `Deleted`, identified by package ID and version.

## Implementation Changes
- Extend publish results and `IndexRunSummary` with ordered collections of added, updated, and deleted package identities.
- In `SqliteIndexStore`, classify:
  - missing existing version as added;
  - existing version with a different content hash as updated;
  - pruned version as deleted;
  - identical hashes as unchanged internally.
- Return deleted package ID and version from pruning instead of only deleting by database ID.
- Keep the database schema and stored `changed_count`/`unchanged_count`; calculate changed count as added plus updated.
- Update `IndexerRunner` output to retain source, status, discovered, indexed, and errors, then print:
  - `Added (n)` with `PackageId Version`
  - `Updated (n)` with `PackageId Version`
  - `Deleted (n)` with `PackageId Version`
- Display `(none)` for empty sections and sort entries deterministically by package ID and version.
- Remove the current `Changed` and `Unchanged` lines from console output.

## Test Plan
- Verify an initial index reports the package version as added.
- Verify changed content for the same package version reports updated, not added or deleted.
- Verify pruning reports each removed package version as deleted.
- Verify an idempotent run produces three empty change sections.
- Verify failed package processing does not incorrectly report the retained version as deleted.
- Add runner logging assertions for names, versions, counts, empty sections, and removal of the old summary lines.
- Run the complete solution build, unit tests, and integration tests.

## Assumptions
- “Library” means a NuGet package version, displayed as package ID plus version.
- Updating a version is one updated entry, even though persistence replaces its stored rows.
- Deletions are reported only for versions actually pruned during a successful publication.
- No database migration or public MCP contract change is required.
