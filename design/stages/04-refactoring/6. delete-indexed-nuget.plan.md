# Delete Indexed NuGet Package

## Summary
Add `Delete` to package JSON configuration. When true, remove every indexed version of that package from its configured environment without contacting the NuGet source.

## Implementation Changes
- Add `bool Delete` to `NuGetPackageOptions`, defaulting to `false`.
- Treat delete entries as tombstones requiring only valid `Environment` and `PackageId`; skip indexing-policy validation.
- Split each environment’s configuration into active package selections and package deletions.
- Include an environment run when it has active packages, delete tombstones, or both.
- Never send delete entries through discovery or download.
- Add explicit package deletion to the index store:
  - scope by source/environment and normalized package ID;
  - remove document FTS entries, all versions, and the library row transactionally;
  - return all removed package/version identities for the `Deleted` report.
- Preserve existing pruning behavior when active packages exist.
- When a source contains only delete tombstones, disable broad missing-version pruning so unrelated indexed packages remain untouched.
- A missing package is a successful no-op. A retained tombstone keeps the package deleted on subsequent runs.
- Keep database schema, exit codes, `Changed` count semantics, and MCP contracts unchanged.

## Reporting
- Report every removed version under `Deleted (n)` using package ID and version.
- A deletion-only run reports zero discovered/indexed packages and succeeds.
- Repeated deletion reports `Deleted (0)` after the first successful removal.

## Test Plan
- Validate and load minimal `{ Delete, Environment, PackageId }` JSON.
- Verify normal package JSON still defaults to `Delete: false`.
- Index multiple versions, mark the package deleted, and verify all versions, search records, and the library row are removed.
- Verify deletion is scoped to the selected environment and does not affect the same package elsewhere.
- Verify a source containing only tombstones does not contact NuGet or prune unrelated packages.
- Verify mixed active and delete entries index active packages and delete tombstoned packages atomically.
- Verify missing and repeatedly deleted packages are successful no-ops.
- Verify the indexer summary lists deleted package versions deterministically.
- Run the full build, unit tests, integration tests, and child-process tests.

## Assumptions
- `Delete: true` deletes all indexed versions of the package in that environment.
- The JSON file remains as a persistent tombstone until removed or changed to `Delete: false`.
- Re-indexing requires changing/removing the tombstone.
- No database migration is required.
